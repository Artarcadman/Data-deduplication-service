### Проект по дедупликации бинарных данных

## Цель проекта

Разработка и реализация прототипа системы дедупликации данных на уровне фиксированных сегментов. Система предназначена для анализа бинарных файлов, выявления повторяющихся сегментов и эффективного хранения метаданных в реляционной базе данных (PostgreSQL).

## Архитектура системы

Проект построен на двухуровневой архитектуре:

1. Обработчик (Python): Отвечает за чтение бинарного файла, нарезку его на фиксированные сегменты (4 байта), вычисление хэша SHA-256 для каждого сегмента и управление логикой "уже обработан/обработать".

2. База данных (PostgreSQL): Используется для постоянного хранения метаданных и реализации логики дедупликации с помощью двух таблиц.

## Схема базы данных

В проекте используются две ключевые таблицы, обеспечивающие нормализацию данных и отказоустойчивость:

Таблица|Назначение

| Таблица | Назначение |
|----------|----------|
|file_chunks|Хранит уникальные сегменты. Главная таблица дедупликации|
|processed_files|Реестр обработанных файлов. Предотвращает повторную обработку всего файла, экономит время и ресурсы.|
